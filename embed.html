Options

    
Add any of these to the same tag.

    

      
AttributeDoes

      
        
data-columnsColumns on desktop. Default 3. Reduced automatically for wide crops, and ignored for a timeline, which always spans the row. Collapses to 2, then 1, on smaller screens.

        
data-limitShow only the newest N cases. One patient counts as one, however many photos they have.

        
data-accentAccent colour. Any CSS colour.

        
data-radiusCorner radius in pixels. Default 2.

        
data-labelsoff hides the BEFORE / AFTER and interval tags.

        
data-captionoff hides the treatment line under each case.

        
data-targetThe id of an element to render into, instead of in place.

        
data-styleoff hands the styling to you. See below.

      
    

  


  

    
What a case looks like

    
Not every case is one before and one after. The gallery reads the clinic's photographs the way the clinic took them, and lays each kind out on its own terms. There is nothing to configure.

    

      
KindShows as

      
        
One patient, tracked over timeA single wide card: the before photo, then each follow-up, labelled 3 months, 12 months, and so on.

        
One result, three viewsOne before/after pair with small tabs to turn the head between frontal and both obliques.

        
One before, one afterExactly that.

      
    

    

      Photographs are also grouped by how tightly they are cropped, because most patients agree to publish only the lower face and a lower-face photo is not the same shape as a full-face one. Each group gets a frame that suits it, so rows stay even instead of ragged. Most clinics settle into one crop and never see this happen.
    

  


  

    
If you want to style it yourself

    
By default the gallery is rendered in a shadow root, so your site's CSS cannot reach it and ours cannot leak into your site. That is deliberate: it means one design survives any theme, and we can improve it without ever touching your code.

    
If you would rather own the design, add data-style="off". We then output plain semantic markup with no styling at all, and you style it however you like. The content still updates itself; only the CSS becomes yours.

    
<script src="https://skinday.com/gallery-embed.js"
        data-clinic-id=""
        data-style="off"></script>

.skinday-gallery           the container
.skinday-gallery__section  one crop group (data-crop)
.skinday-gallery__grid     wraps the cases in a section
.skinday-gallery__case     one card. Also carries __series, so a stylesheet
.skinday-gallery__series   written before still finds its target.
                           (data-mode, data-crop, data-treatment, data-angle)
.skinday-gallery__pair     the before / after pair
.skinday-gallery__strip    a timeline row: before, then each follow-up
.skinday-gallery__figure   one photo (data-when="before" | "after")
.skinday-gallery__img      the image
.skinday-gallery__label    the BEFORE / AFTER or interval caption
.skinday-gallery__tabs     the angle switcher
.skinday-gallery__tab      one angle button (data-angle, aria-selected)
.skinday-gallery__meta     the treatment line
.skinday-gallery__note     the results-vary line
.skinday-gallery__empty    shown when nothing is published

    
data-mode on a card is timeline, angles or pair. The angle tabs keep working in this mode: they are behaviour, not styling, so they stay ours. Everything else is yours.

  


  

    
If you want to build it from scratch

    
Skip the script and read the JSON directly. Public, no key, CORS open.

    
GET https://skinday.com/api/clinic-gallery?clinic_id=

{
  "ok": true,
  "clinic_id": "",
  "count": 2,
  "series_count": 1,
  "cases": [
    {
      "case_id":   "case_4890cb94f1f4c8de",
      "series_id": "ser_2f81a0c47b3ee915",
      "treatment": "biostim",
      "subtype":   "sculptra",
      "angle":     "frontal",
      "crop":      "lower",
      "interval_months": 3,
      "before_date": "Jan 2026",
      "after_date":  "Apr 2026",
      "before_url": "https://.../before.jpg",
      "after_url":  "https://.../after.jpg",
      "created_at": "2026-07-12T01:25:34Z"
    }
  ],
  "series": [
    {
      "series_id": "ser_2f81a0c47b3ee915",
      "mode":      "timeline",
      "treatment": "biostim",
      "crop":      "lower",
      "cases":     [ ... the same two case objects, in order ... ]
    }
  ]
}

    
Two readings of the same data, and nothing is duplicated. cases is the flat list: one entry per stored before-and-after, which is what the clinic's Visualize matches on. series groups the ones that came from the same patient at the same sitting, which is what a visitor is actually looking at. Use whichever fits what you are building. mode is timeline, angles or pair.

    
Only cases the patient consented to publish are ever returned. There is no way to request an unpublished one.
